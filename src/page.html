Что такое стартовый шаблон и зачем он нужен?

Давайте представим обычный процесс подготовки к верстке/разработке сайта. Мы создаем иерархию папок и файлов, настраиваем тот или иной сборщик проекта, подключаем основные JS-дополнения для работы.

А при разработке, каждый раз выполняем одни и те же действия: конвертируем шрифты, оптимизируем картинки, пишем один и тот же код для решения стандартных задач. Ведь меню-бургер, слайдеры, попапы, табы, валидация и отправка форм, спойлеры, галереи и др. давно стали неотъемлемыми частями любого современного сайта.

Так вот, я более 10 лет разрабатываю сайты на фрилансе и успел поработать с самыми разными дизайн-макетами, а также со многими заказчиками и их «хотелками».

По мере решения задач я всячески пытался оптимизировать и автоматизировать рутинные процессы. А самые нужные и удачные HTML/CSS/JS решения составлять в свои чертоги.

Наряду со стандартным функционалом стали появляться и собственные разработки и решения. Например отзывчивый адаптив, динамический адаптив и т.п.

В результате за столько лет добра накопилось достаточно для того, чтобы разработка с использованием ЧФ превратилась в сплошное удовольствие.

Удобная архитектура папок и файлов, автоматизация всех рутинных процессов, не только стандартный JS-функционал, но и решения, позволяющие удовлетворить кастомные уникальные запросы. При этом никакого лишнего кода подключается только нужный функционал для конкретного проекта, что делает результат разработки с ЧФ максимально оптимизированным и быстрым.

Отвечая на вопрос, что такое стартовый шаблон и зачем он нужен, я приведу три основных пункта:

  1. Стартовый шаблон «Мечта верстальщика» — это совокупность оптимизации и автоматизации рутинных процессов при разработке сайтов. База большого количества как стандартных, так и уникальных решений, которые помогут решить множество задач, возникающих в считанные секунды.
  2. ЧФ разработан так, чтобы не притуплять ваши навыки в разработке. Найден баланс между готовыми решениями и полным пониманием пользователем шаблона того, как все устроено под капотом. Также крайне подробное комментирование кода делает из ЧФ обучающий инструмент для развития вас как специалистов
  3. Шаблон «Мечта верстальщика» – работает с разработчиком, а не вместо него!

Возможности «Мечты верстальщика»

Здесь собраны все возможности стартового шаблона, включая возможности инструмента для сборки проекта. Данные будут обновляться по мере добавления функционала.

Инструменты для сборки проекта и их режимы работы

В шаблоне работают два инструмента GULP и WEBPACK, каждый делает то, в чем он хорош. Есть возможность работать в режимах разработки и продакшена. Режим разработки выполняет только необходимые работы, а продакшн обеспечивает оптимизацию файлов, конвертацию и другие необходимые действия. Таким образом, увеличивается скорость работы инструментов. Добавлю, что тип работы указан в package.json равен module

Инструмент GULP

GULP играет роль вспомогательного инструмента и выполняет следующие задачи:

  • Очистка папки с результатом [все режимы]
  • Создает файл.gitignore с указанным содержимым [все режимы]
  • Обрабатывает HTML файлы: добавляет конструкцию с изображением в.WEBP формате, добавляет в подключенные JS CSS файлы версию, что позволяет избежать проблем с кэшированием во время демонстрации работы. [режим продакшена]
  • Обрабатывает CSS файлы: группирует медиазапросы, добавляет вендорные префиксы для обеспечения кроссбраузерности, добавляет подключение. WEBP изображение оптимизирует и сжимает файл. [режим продакшена]
  • Обрабатывает JS файлы: только передает информацию WEBPACK [режим продакшена]
  • Обработка файлов изображений: сжатие и оптимизация форматов.JPG, .PNG, .SVG. На основе.JPG/.PNG файлов формируется.WEBP изображение. [режим продакшена]
  • Создает SVG-спрайт
  • Отправка файлов с результатом на сервер FTP [режим продакшена]
  • Создает ZIP архив с результатом [режим продакшена]

Инструмент WEBPACK

WEBPACK играет основную роль в сборе HTML/SCSS/JS файлов и выполняет следующие задачи:

  • Запускает локальный сервер и открывает HTML-страницы результата в браузере. При внесении изменений сам обновляет страницу. Подключен модуль горячей замены, увеличивающий скорость обновления. Также созданные файлы не пишутся на диске, а находятся в памяти. [режим разработчика]
  • Обрабатывает SCSS файлы: производится преобразование в CSS файлы, переименовываются псевдонимы [все режимы]
  • Обрабатывает HTML файлы: производится сборка файлов из подключаемых частей, переименовываются псевдонимы [все режимы]
  • Обрабатывает PUG файлы: производится сборка файлов из подключаемых частей, переименовываются псевдонимы, производится преобразование в HTML файлы [все режимы]
  • Обрабатывает JS файлы: производится грамотная сборка подключенных модулей, таким образом, в файл с результатом попадает только задействованный в проекте код [все режимы], файл сжимается и оптимизируется [режим продакшена]
  • Копирует файлы из папки files в папку с результатом [все режимы]

Возможности работы с HTML файлами

HTML файлы можно собирать из частей благодаря шаблонизатору FileInclude. То есть, мы можем вынести отдельные части кода, например header и footer, в отдельные файлы и подключать их для всех страниц сайта. Таким образом, при необходимости внести изменения, нам достаточно редактировать только один подключаемый файл. FileInclude позволяет передавать переменные со значением в подключаемый файл. К примеру, мы можем передать свой заголовок (title) для каждой страницы

Использование препроцессора SASS/SCSS, возможности работы со стилями

В шаблоне используется препроцессор SASS в синтаксисе SCSS. Благодаря этому есть возможность обрадовать вас отличными наработками:

  • Отзывчивое свойство. Миксин позволяет задать значение для того или иного свойства, которое будет зависеть от ширины экрана (вьюпорта), что в свою очередь открывает колоссальные возможности для адаптивной (чуткой) верстки.
  • Архитектура подключаемых файлов позволяет использовать только то, что нужно для конкретного проекта.
  • Готовые базовые стили для разных JS модулей

JavaScript возможности, использование модулей

Отметим, что поскольку JS файлы собирает WEBPACK, то появляется возможность использовать подключение ES6 модулей. Таким образом, в результате попадает только используемый функционал. Для добавления той или иной возможности обычно достаточно раскомментировать строчку. Также каждый модуль удобно управляется и настраивается с помощью даты атрибутов в HTML/PUG файлах, а сам код подробно прокомментирован.
